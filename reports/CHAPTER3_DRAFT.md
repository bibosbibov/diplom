### 3.1.2. Метрики качества

Для оценки модели использованы метрики, описанные в разделе 2.3.6:

1. **Macro-F1** — среднее по 12 классификационным головам; компенсирует
   дисбаланс классов.
2. **Vector Accuracy** — доля записей, у которых все 11 базовых метрик
   предсказаны верно (E исключается, поскольку её ground-truth доступен лишь
   у части записей).
3. **Среднее число правильных метрик** — partial accuracy в диапазоне 0…11.
4. **MAE и RMSE по CVSS-баллу** — рассчитываются собственным калькулятором
   CVSS v4.0 (раздел 2.3.5) над предсказанными и истинными метриками.
5. **Severity Accuracy** и **Severity Within ±1** — точность предсказания
   уровня критичности (Critical/High/Medium/Low/None) с учётом упорядоченности
   шкалы.

Итоговые показатели модели на test set (972 v4.0-записи) приведены в

| Показатель | Значение |

Полученные результаты позволяют сделать следующие выводы:

* **Macro-F1 = 0,71** — модель уверенно превосходит случайный baseline и
  устойчиво классифицирует большинство метрик с учётом дисбаланса.
* **Среднее число правильных метрик 9,39 из 11 (85,4%)** — основной
  показатель практической применимости. Это означает, что в среднем
  система ошибается лишь в 1,6 метриках из 11.
* **Severity Within ±1 = 92,1%** — то есть в 92% случаев модель попадает
  либо в истинный уровень критичности, либо в соседний (например,
  High вместо Critical). При принятии оперативных решений по уязвимостям
  это означает, что в 9 случаях из 10 модель не приведёт к качественной
  ошибке в категоризации риска.

Вместе с тем **Vector Accuracy = 39,9%** объяснимо высокой размерностью
задачи: вероятность одновременного попадания во все 11 ортогональных
классов при per-metric accuracy 85% оценивается как ≈ 0,85¹¹ ≈ 0,17, что
согласуется с экспериментальными 0,40 (положительная корреляция между
ошибками говорит о наличии «трудных» записей).

## 3.4. Анализ результатов по группам метрик

Метрики базового вектора CVSS v4.0 концептуально разбиваются на четыре
группы (раздел 1.3 «CVSS v4.0»). Анализ показателей по группам выявляет
характерные паттерны качества модели.

### 3.4.1. Группа Impact на уязвимой системе (VC, VI, VA)

Средний macro-F1 = **(0,789 + 0,820 + 0,795) / 3 ≈ 0,801**.

Это **наиболее точно предсказываемая группа базовых метрик**. Причины:
* импакт на CIA-триаду — самый частый и насыщенный лексическими маркерами
  тип информации в описаниях («denial of service», «code execution»,
  «information disclosure», «утечка», «отказ в обслуживании»);
* объёмная и сбалансированная обучающая выборка (см. рис. 3.1 — гистограммы
  классов: для VC/VI/VA доля мажоритарного класса 39,8–44,0% против
  87,7% у AT и 89,4% у AC).

### 3.4.2. Группа Exploitability (AV, AC, AT, PR, UI)

Средний macro-F1 = **(0,517 + 0,748 + 0,743 + 0,631 + 0,641) / 5 ≈ 0,656**.

Качество ниже из-за двух факторов:

1. **AV (F1 = 0,517)** — самый слабый показатель. При accuracy 90,2% F1
   проседает из-за полного отсутствия предсказаний для редкого класса
   `Physical` (F1 для P = 0,000, поддержка лишь 7 записей в test). Аналогичная
   ситуация с `Adjacent` (F1 = 0,243, support = 60).
2. **PR (F1 = 0,631)** — модель путает уровни привилегий, особенно
   None ↔ Low. В описаниях редко встречается явное указание того, требуется
   ли «обычная учётная запись» или «доступ без аутентификации»;
   контекстная подсказка приходит из CWE (например, CWE-862 «Missing
   Authorization» указывает на PR=N), но этого недостаточно для устойчивого
   различения.

### 3.4.3. Группа Subsequent System (SC, SI, SA)

Средний macro-F1 = **(0,623 + 0,671 + 0,656) / 3 ≈ 0,650**.

Эти метрики введены впервые в стандарте CVSS v4.0 и отражают **влияние на
другие системы вне уязвимой**. Качество ниже, чем у Impact-группы, по двум
причинам:

* в обучающих текстах редко встречаются явные указания на распространение
  атаки за пределы уязвимой системы (мажоритарный класс `N` — 86–89%);
* для классификационных голов SC, SI, SA веса инициализировались случайно
  на этапе 2 (раздел 2.3.4) — в отличие от голов VC/VI/VA, переиспользованных
  из этапа 1. Это объясняет относительно меньший прирост качества.

Тем не менее accuracy 87–89% — приемлемый результат, особенно с учётом
крайне разреженного позитивного сигнала (H/L составляют 10–13% выборки).

### 3.4.4. Метрика Exploit Maturity (E)

F1 = **0,875** — наилучший показатель среди всех. Это объясняется
встроенной поддержкой из числовых признаков:

* признак `exploit` (наличие записи в ExploitDB) — сильный детерминированный
  индикатор класса `A` (Attacked);
* признак `kev` (попадание в каталог CISA KEV) — индикатор класса `A`;
* признак `epss` — корреляция с моделью эксплуатируемости.

Модель эффективно использует эти структурированные признаки через MLP-голову
fusion-слоя (раздел 2.2.5), что и даёт уверенный F1 = 0,875 даже при том,
что поддержка по этой метрике меньше (553 записи: только те, где E явно
указан в исходном векторе).

### 3.4.5. Влияние дисбаланса классов

Анализ распределения классов на train (раздел 2.3.2, рисунок 3.1) показывает
сильный дисбаланс по большинству метрик:

* AC: 89,4% — Low;
* AT: 87,7% — None;
* SC/SI/SA: 86–89% — None;
* UI: 83,2% — None.

Это типичная картина для области ИБ: подавляющее большинство уязвимостей
не требуют физического доступа, дополнительных условий или не оказывают
вторичного воздействия. Multi-task обучение без явного re-weighting
приводит к тому, что **accuracy завышен, но F1 — честный показатель**,
выявляющий, насколько хорошо модель различает миноритарные классы.
Подробный численный разбор дисбаланса — `reports/class_imbalance.md`.

### 3.6.3. Анализ калибровки confidence

На рисунке 3.3 (`reports/figures/confidence_analysis.png`) приведены:

* **(а)** scatter «средняя уверенность → точность» — модель показывает
  монотонную, но недокалиброванную зависимость: при средней уверенности
  0,8 фактическая точность ≈ 0,75–0,80;
* **(б)** распределение softmax-уверенности отдельно для верных и неверных
  предсказаний — пики хорошо разделимы.

Численные значения:

* **Средний confidence на верно предсказанных метриках: 0,905**;
* **Средний confidence на ошибочно предсказанных метриках: 0,725**.

Разница в 0,18 — статистически значимая. Это означает, что **порог
confidence ≈ 0,7 является разумным критерием доверия**: предсказания
выше этого порога обычно правильны, ниже — требуют ручной проверки.

Однако из 972 × 12 = 11 664 предсказаний 7 431 имеют confidence > 0,9, и
**часть из них ошибочны**. Топ-10 «уверенных, но неправильных»
предсказаний приведены в `reports/error_analysis/high_confidence_errors.csv`.
Эти случаи представляют интерес для дальнейшего исследования — модель
здесь работает «слепо», без сигнала неопределённости.

### 3.6.4. Анализ ошибок по уровням severity

Матрица переходов `true → pred` для уровня критичности приведена в
`reports/severity_transitions.csv` и в виде heatmap — в
`reports/figures/severity_confusion_matrix.png`.

**Распределение типов ошибок:**

| Тип | Количество | Доля |
|:--|--:|--:|
| Точное совпадение | 655 / 972 | **67,4%** |
| Завышение (overestimate) | 153 / 972 | 15,7% |
| Занижение (underestimate) | 164 / 972 | 16,9% |

**Топ-5 крупнейших ошибочных переходов:**

| true → pred | Количество | Тип |
|:--|--:|:--|
| Medium → High | 65 | завышение |
| High → Medium | 48 | занижение |
| High → Critical | 42 | завышение |
| Critical → High | 37 | занижение |
| Medium → Low | 33 | занижение |

**Закономерность.** Завышения и занижения примерно симметричны (15,7% и

показателем Severity Within ±1 = 92,1%.

**Практический смысл:** для оперативного приоритизирования уязвимостей
модель пригодна — даже при ошибке она крайне редко (≤3%) перебрасывает
запись через две и более ступени критичности.

## 3.7. Анализ показательных примеров

В приложении к ВКР приведены 5 разобранных примеров предсказаний
(файл `reports/showcase_examples.md`). Здесь даны краткие характеристики.

### Пример 1 (идеальное предсказание)
CVE-2024-6115 / CWE-434 (Unrestricted File Upload). 11/11 метрик
предсказаны верно, итоговый score 6,9 / Medium совпадает с истинным.
Средняя уверенность модели — 0,955. Описание содержит классические
маркеры («unrestricted upload», «launched remotely»).

### Пример 2 (идеальное предсказание)
CVE-2024-12928 / CWE-74 (Injection). 11/11. Score 5,3 / Medium.
Уверенность 0,961. Аналогичный паттерн: чёткий тип уязвимости + явный
лексический маркер «sql injection».

### Пример 3 (частичная ошибка, 9/11)
CVE-2025-9235 / CWE-79 (XSS). Score завышен до 5,5 (true 2,0).
Ошибки: **PR** (true=L → pred=N, conf=0,53) и **UI** (true=P → pred=N,
conf=0,56). Обе ошибочные метрики имеют низкую уверенность модели —
ниже порога 0,7. Это пример **корректного поведения сигнала
неопределённости**: модель сигнализирует, что не уверена в этих
конкретных метриках.

### Пример 4 (частичная ошибка, 8/11)
CVE-2026-21989 / CWE-20 (Improper Input Validation). Критичная ошибка:
модель проставила VC=N/VI=N/VA=N, в то время как истинные значения
H/H/L. Причина — крайне общее описание («недостаточная проверка
входных данных… может повлиять на чтение, изменение или удаление»),
без явных лексических маркеров CIA-импакта.

### Пример 5 (грубая ошибка, 5/11)
CVE-2024-52276 / CWE-451 (UI Misrepresentation). Severity угаданa
(High = High), но 4 метрики ошибочны, включая AV (L → N) и AT (N → P,
conf=0,90 — высокая уверенность при ошибке). Это пример случая, когда
модель не понимает специфической семантики «локальное отображение
информации» и применяет шаблон «удалённой атаки на веб».

   подтверждают пригодность системы для автоматизированной оценки
   критичности уязвимостей.

2. **Severity Within ±1 = 92,1%** делает систему практически применимой
   для оперативного приоритизирования: ошибки «на две и более ступени»
   составляют менее 3% случаев.

3. **Стабильность val vs test** (среднее |Δ| F1 = 0,016) подтверждает
   отсутствие переобучения и воспроизводимость качества на независимой
   выборке.

4. **Качество по группам метрик** иерархично:
   * Impact (VC/VI/VA) — лучший (F1 ≈ 0,80);
   * Exploit Maturity (E) — лучший в целом (F1 = 0,88) благодаря
     структурированным признакам;
   * Subsequent (SC/SI/SA) и часть Exploitability (PR/UI) — слабее
     (F1 ≈ 0,63), что отражает разреженность сигнала и дисбаланс
     классов;
   * AV — самая слабая (F1 = 0,52) из-за фактической невозможности
     различить миноритарные классы Physical и Adjacent.

5. **Калибровка confidence.** Разница среднего confidence на верных и
   неверных предсказаниях составляет 0,18 (0,905 против 0,725), что
   делает softmax-уверенность пригодной для механизма «доверять/проверить
   вручную». Порог 0,7 разделяет эти распределения с приемлемой
   полнотой.

6. **Профиль ошибок по типам уязвимостей** показывает, что модель
   эффективна для уязвимостей с чёткими лексическими маркерами
   (инъекции, переполнения буфера) и слабее для семантически размытых
   типов (Improper Access Control, Path Traversal). Это направление
   для дальнейших улучшений.

7. **Симметричность ошибок severity** (16% завышений / 17% занижений)
   подтверждает отсутствие систематического смещения модели и
   корректность калькулятора CVSS v4.0 на интервале 0–10.

Полученные результаты подтверждают, что предложенная архитектура mBERT с
fusion-слоем для числовых признаков и 12 классификационными головами,
обученная двухэтапной стратегией с предобучением на CVSS v3.1,
обеспечивает приемлемое для практического применения качество оценки
критичности уязвимостей по CVSS v4.0.
